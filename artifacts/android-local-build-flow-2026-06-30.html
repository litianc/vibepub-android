VibePub Android 开发闭环从“等发布”改成“真机快速验证”

这张图记录 2026-06-30 的流程改造：Android 可以在本机完成编译、安装和真机自动化；上传后的云端处理仍由 Cloudflare Worker、D1、R2 和 GitHub Actions mining job 协作完成。

分支：codex/android-experience-v1
设备策略：红米真机优先
后端：Cloudflare + GitHub Actions

为什么这次流程会变快

之前：每轮都等线上 APK

适合发布确认
1
本地改代码Android 或 Worker 修改完成后不能马上上机看效果。
2
提交并推送必须走 GitHub Actions 之后才有新 APK。
3
远端排队打包runner 启动、拉代码、跑测试、生成 Release。
4
下载再安装回到 Mac 下载 APK，然后手动或 ADB 安装。
5
再看真机结果一次小改动也会被远端打包周期拉长。

现在：本地先闭环，线上做共享

适合日常迭代
1
本地测试scripts/build-android-local.sh test 固定使用 JDK 21，避免 Robolectric 被外层 JDK 影响。
2
本地打包scripts/build-android-local.sh assemble 生成 app-debug.apk。
3
ADB 安装到真机scripts/install-android-local-apk.sh 可测试、打包、安装、启动一条命令完成。
4
录音上传后自动触发处理Cloudflare Worker 收到上传后点名触发 mining job，不再只等定时扫描。
5
自动化收证据真机 smoke 保存截图、UI dump、logcat、mining run 和 audit 结果。

系统协作泳道

Mac 本地开发
Android 代码与单测

Compose、Room、WorkManager、Media3 的体验改动先在本机验证。

build
本地 APK

输出 android/app/build/outputs/apk/debug/app-debug.apk，不必等待 Release。

install
ADB 脚本

安装、启动、权限、导入测试音频和视觉 smoke 都由脚本串起来。

红米真机
真实 App 体验

首页状态、录音详情、播放、复制、分享、诊断都在真机上看。

upload
录音文件

测试音频模拟用户说话，目标是一条非零秒记录，不出现重复零秒项。

audit
视觉证据

截图、UI XML、logcat 与 audit markdown 保存到 artifacts/。

Cloudflare 服务
Worker API

/api/uploads 接收录音，/api/recordings 返回状态、时长、标题和错误信息。

store
R2 + D1

R2 保存音频文件，D1 保存记录状态、duration_ms 和处理进度字段。

dispatch
点名触发

Worker 带 target_filename 触发 GitHub Actions，减少等待下一次扫描。

GitHub Actions
Mining Job

读取目标录音，执行 ASR、文章生成和公众号草稿链路。

sync
状态回写

完成后端可被 App 同步为处理中、完成或失败，错误原因进入可诊断路径。

release
Release APK

仍保留作为团队共享安装包和发布记录，不再阻塞每次开发验证。

速度与证据

~33s Android 全量单测

本机 JDK 21 环境下，体验改动后可快速回归。

~16s 增量 assemble

已有构建缓存时，本地生成 debug APK 的等待明显缩短。

1 条 目标录音处理

Worker dispatch 带 target_filename，mining job 优先处理本次上传。

4 类 真机证据

截图、UI dump、logcat、audit 结果组成 dogfood 回归材料。

日常命令入口

# 本地测试和打包
scripts/build-android-local.sh test
scripts/build-android-local.sh assemble

# 本地 APK
android/app/build/outputs/apk/debug/app-debug.apk
# 安装到已连接真机
scripts/install-android-local-apk.sh --serial <adb-serial> --test

# 完整 dogfood E2E
scripts/run-android-device-smoke.sh \
  android/app/build/outputs/apk/debug/app-debug.apk
