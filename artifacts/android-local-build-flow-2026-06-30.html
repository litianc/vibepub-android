Android 迭代从“等云端打包”改成“本地快速闭环”

目标是把 VibePub Android 的日常开发循环缩短：本地编译、本地安装、真机验证；只有需要共享 APK 或做发布边界确认时，才走 GitHub Release。

记录：2026-06-30 · 分支 codex/android-experience-v1

新旧流程对比

旧流程：每次都等 GitHub

慢 · 适合发布
1
修改代码本地完成改动。
2
提交并推送必须把代码推到 GitHub。
3
Actions 排队runner 启动、拉代码、装 SDK。
4
云端测试与打包生成 Release APK。
5
下载再安装回到本机用 ADB 装到平板。

新流程：本地先闭环

快 · 适合迭代
1
修改代码Android UI、Worker 契约、测试都在本地改。
2
本地测试gradle -p android :app:testDebugUnitTest
3
本地打包gradle -p android :app:assembleDebug
4
ADB 安装adb install -r app-debug.apk，直接到红米平板。
5
真机验证截图、UI dump、logcat、audit 结果保存在 artifacts。

基础设施分工

SDK

本机最小 SDK

只安装 command line tools、Android 36 platform、build-tools 36，不装 Android Studio 和模拟器。

ADB

真机优先

红米平板通过 ADB 做安装、启动、权限、导入音频和视觉验证。

GH

Release 只做分发

需要给人 APK 链接、固定 dogfood 版本、或做远端确认时再触发 GitHub Build & Release。

DOC

证据同步

MANIFEST 和 progress artifact 记录当前 APK、测试、安装和待验证边界。

命令入口

# 一次性安装最小 Android SDK 管理工具
brew install --cask android-commandlinetools

# 本机 SDK 根目录
export ANDROID_HOME=/opt/homebrew/share/android-commandlinetools
export ANDROID_SDK_ROOT=$ANDROID_HOME

# 接下来补齐项目所需 platform/build-tools
yes | sdkmanager --licenses
sdkmanager "platforms;android-36" "build-tools;36.0.0"
# 日常快速闭环
scripts/build-android-local.sh test
scripts/build-android-local.sh assemble
scripts/install-android-local-apk.sh --serial 10.161.2.96:43355 --skip-build

# 或者一条命令完成测试、打包、安装、启动
scripts/install-android-local-apk.sh --serial 10.161.2.96:43355 --test
